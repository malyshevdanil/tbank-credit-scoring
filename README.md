#  Кредитный скоринг / Тинькофф Банк

ML-модель для предсказания вероятности дефолта клиента, подключённая к REST API и веб-интерфейсу.

---

##  Идея проекта

Из полного датасета намеренно отобраны 5 простых признаков. Цель: смоделировать акционную форму быстрой подачи заявки, где клиент заполняет минимум полей и за 30 секунд получает предварительное решение по кредиту.

---

## 📊 Данные

| Признак | Описание | Значения |
|---|---|---|
| age | Возраст клиента | Число |
| income | Доход (тыс. руб./мес.) | Число |
| education | Образование | 0 / 1 |
| work | Занятость | 0 / 1 |
| car | Автомобиль | 0 / 1 |
| default | Дефолт (целевая переменная) | 0 / 1 |

Датасет: **180 816 строк**, разбивка 80/20 (train/test)

##  Модели и результаты

Сравнивались три модели с применением `class_weight='balanced'` из-за несбалансированности датасета:

| Метрика | Логрегрессия | Random Forest | XGBoost |
|---|---|---|---|
| Precision | 14% | 14% | 14% |
| Recall | 67% | 47% | 71% |
| Accuracy | 50% | 61% | 45% |

### Финальная модель: Логистическая регрессия + StandardScaler

| Метрика | Результат |
|---|---|
| Precision | 15% |
| Recall | 67% |
| Accuracy | 50% |

Выбор в пользу логистической регрессии обусловлен тремя факторами. Во-первых, она показала наилучшую полноту (67%), что важнее точности в задаче кредитного скоринга: банку критичнее поймать реального неплательщика, чем минимизировать ложные отказы. Во-вторых, модель легко интерпретируется, что позволяет объяснить причину отказа. В-третьих, логистическая регрессия является индустриальным стандартом для подобных задач.

Невысокие значения precision и accuracy объясняются намеренным ограничением до 5 признаков, несбалансированностью датасета и линейной природой модели. Это осознанный компромисс между удобством клиента и качеством предсказания.

##  Запуск

**1. Установка зависимостей**
```bash
pip install fastapi uvicorn scikit-learn xgboost joblib pandas numpy
```

**2. Запуск API**
```bash
uvicorn service:app --reload
```

**3. Открыть фронтенд**

Открыть `index.html` в браузере. API доступен на `http://localhost:8000`

---

##  API

**POST** `/score`

##  Интерфейс

Клиент заполняет форму из 5 полей и получает решение:
- ✅ **Кредит одобрен** — высокая вероятность одобрения
- ❌ **Кредит отклонён** — рекомендуем улучшить кредитную историю
